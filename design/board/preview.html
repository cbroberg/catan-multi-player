9. Implementation Notes

Hex geometry: All hexes use flat-top orientation with points at 50,0 93.3,14.4 93.3,72.2 50,86.6 6.7,72.2 6.7,14.4 in a 100x115 viewBox. This matches the flat-top hex math from Red Blob Games (60-degree angles).


Player color swapping: In settlement.svg, city.svg, and road.svg, search for gradient stops containing #ef4444 (primary), #dc2626 (mid), #b91c1c (dark), #991b1b (darkest), and #fca5a5 (highlight). Replace these with the equivalent shades of the player's color.


Number token rendering: In the game, tokens should be rendered as React SVG components with dynamic text content and pip count. The number-token.svg file is a visual reference; the actual component should accept number and isRed props.


Accessibility: Each terrain hex has distinct visual patterns beyond color: trees (forest), sheep+fence (pasture), wheat stalks (fields), brick pattern+kiln (hills), snow-capped peaks (mountains), sand dunes+cracks (desert), waves (sea), river+sparkles (gold). These patterns remain visible at all scales.


File sizes: All SVGs are well under 5KB. No raster images, no external dependencies.


Harbor marker: The sign text (ratio + resource icon) should be rendered dynamically in React. The SVG provides the wooden frame design template.
